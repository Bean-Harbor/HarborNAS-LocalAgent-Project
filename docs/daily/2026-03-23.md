# 2026-03-23 联调日志

## 目标

对 HarborOS (http://192.168.3.172/) 进行 **纯 Rust 二进制** 实时联调。

## 关键发现

- REST API (`/api/v2.0/service/*`) 在这台 HarborOS 上返回 **404**，不可用。
- **WebSocket middleware API** (`ws://host/websocket`) 可用且是官方主路径。
- TrueNAS middleware 的 `service.query` filter 参数格式需 **双层数组**: `[[["field","=","value"]], {options}]`。

## 代码变更

1. **`Cargo.toml`**: 新增 `tungstenite = "0.26"` WebSocket 依赖。
2. **`src/orchestrator/executors/harbor_ops.rs`**: 新增 `MiddlewareWsExecutor`。
   - 连接 `ws://host/websocket`，`auth.login` 认证，调 `service.query` / `service.control`。
   - 实现 `Executor` trait，路由为 `MiddlewareApi`（最高优先级）。
3. **`src/main.rs`**: 当 `--harbor-url` + `--harbor-user` + `--harbor-password` 时使用 WS executor；`--harbor-api-key` 时仍走 HTTP REST。
