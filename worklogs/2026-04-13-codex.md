# 2026-04-13 Codex

## 今日目标

- 收口 PR review 中指出的管理面安全问题
- 确保飞书配置页和状态接口不会再泄露敏感信息
- 强化本地直连预览入口，避免被反向代理头部绕过

## 今日完成

### 1. `/api/state` 不再返回敏感字段

已对管理后台状态快照增加脱敏逻辑：

- `feishu_bot.app_secret` 不再出现在接口响应中
- `defaults.rtsp_password` 不再出现在接口响应中

这样桌面端 Web UI 仍可读取状态，但不会把真实密钥和摄像头密码直接暴露给任意访问者。

### 2. 手机配置页不再回显已保存的飞书密钥

已调整 `/setup/mobile`：

- `App Secret` 输入框默认留空
- 页面文案明确说明“出于安全原因不会回显历史 secret”
- 仍保留当前 `App ID` 与绑定状态显示，方便用户确认当前设备绑定的是哪一个 Bot

这避免了“二维码被他人扫到后直接看见已保存 secret”的风险。

### 3. 本地管理面与原始预览入口加上更严格的直连校验

已把以下接口统一纳入“仅本机/局域网直连访问”约束：

- `/api/state`
- `/setup/mobile`
- 绑定二维码与绑定刷新接口
- 飞书配置、默认参数、手动添加摄像头、局域网扫描等管理接口

同时补充了对以下头部的拦截：

- `Forwarded`
- `X-Forwarded-For`
- `X-Forwarded-Host`
- `X-Forwarded-Proto`
- `X-Real-Ip`

这样即便请求最终落到本机端口，也不能通过“带转发头的反向代理/隧道”伪装成本地访问。

### 4. 增补回归测试

已新增并通过测试：

- 转发头存在时，本地管理接口与本地摄像头预览接口都会拒绝访问
- 飞书 Bot 配置脱敏后不再包含 `app_secret`
