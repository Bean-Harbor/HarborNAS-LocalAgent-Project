## IM 协作说明

IM 部分后续明确按分工协作推进，不应继续由单仓库单侧包办。

建议拆分如下：

- 工程师 A：`OpenSource/IM`
  - 各 IM adapter
  - gateway runtime
  - route registry
  - 平台凭据管理
  - 入站归一化与出站投递
- 工程师 B：`HarborNAS-LocalAgent-Project`
  - `assistant_task_api`
  - 业务状态机
  - approval / artifact / audit
  - `POST /api/tasks` 的 IM 契约收口
  - `POST /api/notifications/deliveries` 的 HarborNAS 调用侧

协作边界要求：

- 两边只通过 HTTP/JSON contract 通信
- 不互相 import 代码
- 不共享 `.harbornas/*.json`
- HarborNAS 持有业务会话真源
- IM Gateway 持有平台凭据、路由与传输运行态
